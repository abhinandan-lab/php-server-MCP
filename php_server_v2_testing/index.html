üìÅ Project Architecture

  • /api - Framework root with routing and bootstrapping
  • /api/controllers - All controller files with business logic entry points
  • /api/services - Service classes for clean, testable business logic
  • /api/repository - Database access layer with repository pattern
  • /api/helpers - Reusable helper functions auto-loaded globally
  • /api/middleware - Security, validation, and authentication middleware
  • /api/core - Framework core files (Router, ExceptionHandler, Config)
  • /api/exceptions - Custom exception classes for error handling
  • /api/database/migrations - Database migration files
  • /api/logs - Application logs and error tracking
